# Configuration File (`config.yaml`)

The `config.yaml` file is the central point for configuring the application's behavior, particularly how it parses and interprets `autoinst-log.txt` files from openQA jobs.
By default the application use file configure.yaml in the current folder.
Location of a different configuratio file can be changed using env variable OQTV_CONFIG_FILE

## Top-Level Parameters

These parameters are defined at the root of the `config.yaml` file.

| Parameter               | Status   | Description                                                                                                                                 | Example      |
| ----------------------- | -------- | ------------------------------------------------------------------------------------------------------------------------------------------- | ------------ |
| `config_ver`            | Optional | A version number for the configuration file format. This is currently unused by the application but can be useful for tracking changes.       | `1`          |
| `max_jobs_to_explore`   | Optional | The maximum number of related openQA jobs to discover and fetch, starting from the initial URL. Defaults to `10` if not specified.          | `10`         |
| `cache`                 | Optional | A block containing settings for the caching mechanism. See the `Cache Configuration` section below for details.                               | `(see below)` |
| `autoinst_parser`       | Optional | A list of parser configurations. This is the most important section, defining how logs are processed. See the `Parser Configuration` section. | `(see below)` |

---

## Cache Configuration

The `cache` block contains parameters for controlling the application's caching layer, which stores downloaded assets to speed up subsequent requests.

| Parameter          | Status   | Description                                                                                             | Example      |
| ------------------ | -------- | ------------------------------------------------------------------------------------------------------- | ------------ |
| `cache_dir`        | Optional | The directory where cache files will be stored. Defaults to `./.cache` if not specified.                | `'.cache'`   |
| `cache_max_size`   | Optional | The maximum size of the cache in bytes. If the cache exceeds this size, older files may be removed.     | `1073741824` |

---

## Parser Configuration (`autoinst_parser`)

This is a list of parser objects. The application iterates through this list and uses the **first** parser whose `match_name` regex matches the openQA job name. This allows you to define different parsing rules for different types of tests.

### Parser Object

Each object in the `autoinst_parser` list defines a distinct set of rules.

| Parameter      | Status    | Description                                                                                                                                                                                          | Example                                          |
| -------------- | --------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------ |
| `name`         | Mandatory | A unique name for the parser. Used for logging and identification.                                                                                                                                   | `'SAPHanaSR'`                                    |
| `match_name`   | Mandatory | A regular expression used to associate a job with this parser. It **must** contain a named capture group `(?P<name>...)` which is used to extract a short, human-readable name for display in the UI. | `'.*(?P<name>SAPHanaSR-ScaleUp-PerfOpt).*'` |
| `channels`     | Mandatory | A list of channel objects that define the specific log lines to extract as events. See the `Channel Object` section below.                                                                           | `(see below)`                                    |
